## How to Use This Documentation

### Scenario-Based Guide

#### Scenario 1: "I'm setting this up for the first time"

**Path to follow**:
1. Read [INSTALLATION.md](INSTALLATION.md) completely
2. Follow installation steps for your operating system
3. Verify installation
4. Read [QUICK_START.md](QUICK_START.md) sections 1-3
5. Try "Test Run" from [INSTALLATION.md](INSTALLATION.md)
6. If successful, proceed to [QUICK_START.md](QUICK_START.md) usage examples

**Estimated time**: 1-2 hours (including installation)

---

#### Scenario 2: "I just need to run a simple test"

**Path to follow**:
1. Read [QUICK_START.md](QUICK_START.md) - "Daily Use" section
2. Follow the "3 Steps" to launch software
3. Use the instrument-specific example in [QUICK_START.md](QUICK_START.md)
4. Reference [USAGE_WORKFLOWS.md](USAGE_WORKFLOWS.md) for your specific test type

**Estimated time**: 15-30 minutes (first time), 5 minutes (subsequent uses)

---

#### Scenario 3: "I need to understand voltage ramping"

**Path to follow**:
1. Read [README.md](README.md) - "Advanced Features" → "Automated Voltage Ramping"
2. Read [POWER_SUPPLY_GUIDE.md](POWER_SUPPLY_GUIDE.md) - "Voltage Ramping" section
3. Try [USAGE_WORKFLOWS.md](USAGE_WORKFLOWS.md) - "Workflow 3: Automated Voltage Stress Test"
4. Reference [POWER_SUPPLY_GUIDE.md](POWER_SUPPLY_GUIDE.md) - "Ramping Example"

**Estimated time**: 45 minutes

---

#### Scenario 4: "Something isn't working"

**Path to follow**:
1. Identify symptom category:
   - Installation problem → [INSTALLATION.md](INSTALLATION.md) troubleshooting
   - Connection problem → [USAGE_WORKFLOWS.md](USAGE_WORKFLOWS.md) - "Debugging Connection Issues"
   - Measurement problem → [README.md](README.md) or [POWER_SUPPLY_GUIDE.md](POWER_SUPPLY_GUIDE.md) troubleshooting
   - General usage question → [QUICK_START.md](QUICK_START.md) - "Common Questions"

2. Follow systematic troubleshooting procedure
3. If unresolved, contact support (see [README.md](README.md) - "Getting Help")

**Estimated time**: 15-60 minutes depending on issue

---

#### Scenario 5: "I need to train someone"

**Path to follow**:
1. **Preparation**: Read [README.md](README.md) completely yourself
2. **Installation Session** (if needed):
   - Follow [INSTALLATION.md](INSTALLATION.md) together
   - Verify successful installation
3. **Training Session 1** (1 hour): Basic operation
   - Go through [QUICK_START.md](QUICK_START.md) together
   - Do hands-on exercises
   - Practice daily startup/shutdown
4. **Training Session 2** (1 hour): Instrument-specific
   - Choose relevant instrument
   - Use [README.md](README.md) user guide section
   - Practice real tests
5. **Training Session 3** (Optional, 1 hour): Advanced features
   - Use [USAGE_WORKFLOWS.md](USAGE_WORKFLOWS.md)
   - Practice workflows relevant to their work

**Estimated time**: 3-4 hours total training

## Navigation Tips

### Within Documents

All Markdown documents include:
- **Table of Contents**: Jump to specific sections
- **Cross-references**: Links to related sections
- **Headers**: Hierarchical organization
- **Code blocks**: Formatted commands and examples
- **Tables**: Organized comparison data
- **Examples**: Highlighted with boxes or special formatting

### Between Documents

Use cross-references to navigate:
- Click links to jump between documents
- Use "Back" in your browser to return
- Keep multiple tabs open for reference
- Bookmark frequently accessed sections

### Search Function

In your Markdown reader or text editor:
- Press `Ctrl+F` (Windows/Linux) or `Cmd+F` (Mac)
- Search for keywords
- Examples:
  - "voltage ramping" - Find all mentions
  - "NPLC" - Find technical term usage
  - "error" - Find troubleshooting sections
